# 贡献指南\n\n感谢你对ncmdump-Android项目的兴趣！本文档提供了参与项目开发的相关指南。\n\n## 行为准则\n\n- 友好和尊重地对待所有参与者\n- 欢迎不同观点和建议\n- 关注解决问题而不是争论\n- 接受批评性反馈\n- 专注于对社区最有利的观点\n\n## 如何贡献\n\n### 报告错误 (Bug Reports)\n\n在提交bug报告前，请检查issue列表以确保问题尚未被报告。\n\n当提交bug报告时，请包含以下信息:\n- 清晰简洁的标题和描述\n- 复现步骤\n- 实际行为和预期行为\n- 屏幕截图或日志输出\n- 你的环境信息:\n  - 操作系统和版本\n  - Android版本\n  - 应用版本\n  - Java版本\n\n### 提交功能请求\n\n功能请求欢迎！请提供:\n- 功能的清晰描述\n- 使用场景和为什么这个功能有用\n- 可能的实现方法\n- 相关的第三方库或参考\n\n### 提交代码\n\n1. **Fork项目**\n   ```bash\n   # 在GitHub上fork项目\n   ```\n\n2. **克隆你的fork**\n   ```bash\n   git clone https://github.com/YOUR_USERNAME/ncmdump-Android.git\n   cd ncmdump-android\n   ```\n\n3. **创建特性分支**\n   ```bash\n   git checkout -b feature/amazing-feature\n   # 或修复bug\n   git checkout -b fix/bug-description\n   ```\n\n4. **进行更改**\n   - 按照项目的编码风格进行修改\n   - 添加或更新测试\n   - 更新相关文档\n\n5. **测试你的更改**\n   ```bash\n   # 运行Android测试\n   ./gradlew test\n   ./gradlew lint\n   \n   # 测试前端更改\n   cd frontend\n   npm run build\n   ```\n\n6. **提交你的更改**\n   ```bash\n   git add .\n   git commit -m \"Add: 功能描述\" \n   # 或\n   git commit -m \"Fix: bug描述\"\n   # 或\n   git commit -m \"Docs: 文档更新\"\n   ```\n   \n   提交信息格式:\n   - `Add:` 添加新功能\n   - `Fix:` 修复bug\n   - `Refactor:` 代码重构\n   - `Docs:` 文档更新\n   - `Test:` 添加或更新测试\n   - `Chore:` 构建、依赖等变更\n\n7. **Push到你的fork**\n   ```bash\n   git push origin feature/amazing-feature\n   ```\n\n8. **打开Pull Request**\n   - 在GitHub上打开PR\n   - 提供清晰的PR标题和描述\n   - 关联相关的issue (如果有)\n   - 截图或演示新功能\n\n## 代码风格指南\n\n### Kotlin和Java\n- 遵循[Kotlin官方风格指南](https://kotlinlang.org/docs/coding-conventions.html)\n- 使用4空格缩进\n- 变量名使用camelCase\n- 类名使用PascalCase\n- 添加有意义的注释\n- 函数长度应保持合理(建议<50行)\n\n### TypeScript/React\n- 遵循[Google TypeScript风格指南](https://google.github.io/styleguide/tsguide.html)\n- 使用2空格缩进\n- 组件名称使用PascalCase\n- 变量和函数使用camelCase\n- 提供类型注解\n- 使用功能性组件和hooks\n\n### 通用规则\n- 每行最大长度120个字符\n- 使用有意义的变量名和函数名\n- 删除未使用的导入和代码\n- 添加适当的错误处理\n\n## Pull Request流程\n\n1. **检查清单**\n   - [ ] 代码遵循项目的编码风格\n   - [ ] 自我审查了代码\n   - [ ] 添加了注释来解释复杂逻辑\n   - [ ] 更新了相关文档\n   - [ ] 没有新增警告\n   - [ ] 添加了适当的测试\n   - [ ] 测试通过\n\n2. **审查过程**\n   - 至少需要一个维护者的approval\n   - 可能需要进行更改并重新请求审查\n   - 所有对话应保持建设性\n\n3. **合并**\n   - 一旦获得approval，PR将被合并\n   - 分支会被删除\n\n## 文档贡献\n\n文档改进总是受欢迎的！你可以:\n- 修复拼写和语法错误\n- 改进现有文档的清晰度\n- 添加新的教程或指南\n- 翻译文档\n\n文档以Markdown格式维护，位于项目根目录。\n\n## 开发工具和资源\n\n- [Android Studio](https://developer.android.com/studio)\n- [Android官方文档](https://developer.android.com/docs)\n- [React文档](https://react.dev/)\n- [Vite文档](https://vitejs.dev/)\n- [Kotlin文档](https://kotlinlang.org/docs/)\n\n## 社区标准\n\n我们致力于提供一个安全、包容的环境。所有参与者应:\n- 尊重他人\n- 接受建设性的批评\n- 专注于对社区最有利的讨论\n- 如果遇到冒犯性内容，请报告\n\n## 问题或困惑?\n\n如果你有任何问题:\n1. 查看现有的discussions\n2. 创建新的issue进行讨论\n3. 通过邮件联系: 1250320117@zust.edu.cn\n\n感谢你的贡献！\n"
