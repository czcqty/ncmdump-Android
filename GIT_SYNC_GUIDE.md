# Git同步到个人仓库 - 详细步骤\n\n项目文档已完成准备。现在按以下步骤将项目同步到GitHub个人仓库。\n\n## 前置准备\n\n1. **安装Git**\n   - 如果尚未安装，从 https://git-scm.com/download/win 下载Git for Windows\n   - 安装时选中\"Git Bash Here\"选项\n   - 安装完成后重启终端\n\n2. **验证Git安装**\n   ```powershell\n   git --version\n   ```\n\n3. **配置GitHub身份验证**\n   \n   方法A: 使用SSH密钥 (推荐)\n   ```powershell\n   ssh-keygen -t ed25519 -C \"1250320117@zust.edu.cn\"\n   # 密钥将保存到 ~/.ssh/id_ed25519\n   # 将公钥添加到GitHub: https://github.com/settings/keys\n   ```\n   \n   方法B: 使用Personal Access Token\n   1. 访问 https://github.com/settings/tokens\n   2. 生成新token (repo全选)\n   3. 复制token备用\n\n## 同步步骤\n\n### 步骤1: 初始化本地仓库\n\n```powershell\n# 进入项目目录\ncd d:\\1pt5file\\program\\Android\\ncmdump\\ncmdump-android\n\n# 初始化git仓库\ngit init\n\n# 配置用户信息\ngit config user.name \"czcqty\"\ngit config user.email \"1250320117@zust.edu.cn\"\n\n# 验证配置\ngit config --list\n```\n\n### 步骤2: 添加所有文件到暂存区\n\n```powershell\n# 添加所有文件\ngit add .\n\n# 查看已暂存的文件 (可选)\ngit status\n```\n\n### 步骤3: 创建初始提交\n\n```powershell\ngit commit -m \"Initial commit: restructured ncmdump-Android project with new documentation and build configuration\"\n```\n\n### 步骤4: 添加远程仓库\n\n```powershell\n# 添加origin远程\ngit remote add origin https://github.com/czcqty/ncmdump-Android.git\n\n# 验证远程配置\ngit remote -v\n```\n\n### 步骤5: 重命名主分支为main (如果需要)\n\n```powershell\n# 查看当前分支\ngit branch\n\n# 如果是master，重命名为main\ngit branch -M main\n```\n\n### 步骤6: 推送到远程仓库\n\n```powershell\n# 首次推送 (--force 会覆盖远程所有内容)\ngit push -u origin main --force\n\n# 如果使用SSH\n# git push -u origin main --force --no-verify\n```\n\n## 验证推送成功\n\n1. 访问 https://github.com/czcqty/ncmdump-Android\n2. 检查是否显示:\n   - 最新版本的app文件夹\n   - 最新版本的frontend文件夹\n   - README.md和其他文档\n   - 正确的提交历史\n\n## 后续操作（可选）\n\n### 保护主分支 (在GitHub上)\n1. 进入仓库 Settings → Branches\n2. 在\"Branch protection rules\"中添加`main`分支\n3. 启用以下选项:\n   - Require pull request reviews before merging\n   - Include administrators\n   - Require status checks to pass before merging\n\n### 添加预提交钩子 (可选，用于检查代码质量)\n\n```powershell\n# 如果使用Husky和Lint-staged\ncd frontend\nnpm install husky lint-staged --save-dev\nnpx husky install\n```\n\n### 设置CI/CD (可选)\n\n在项目根目录创建 `.github/workflows/android.yml`:\n\n```yaml\nname: Android CI\n\non:\n  push:\n    branches: [ main ]\n  pull_request:\n    branches: [ main ]\n\njobs:\n  build:\n    runs-on: ubuntu-latest\n    steps:\n    - uses: actions/checkout@v3\n    - name: Set up JDK\n      uses: actions/setup-java@v3\n      with:\n        java-version: '11'\n    - name: Build APK\n      run: ./gradlew assembleDebug\n    - name: Upload APK\n      uses: actions/upload-artifact@v3\n      with:\n        name: app-debug\n        path: app/build/outputs/apk/debug/\n```\n\n## 常见问题\n\n### Q: \"fatal: Could not read from remote repository\"\nA: 检查GitHub凭证配置，确保SSH密钥或token有效\n\n### Q: \"Everything up-to-date\" 但nothing was pushed\nA: 使用 `--force` 标志: `git push origin main --force`\n\n### Q: 本地有未暂存的更改\nA: 运行 `git status` 查看，然后 `git add .` 和 `git commit`\n\n## 后续开发\n\n完成同步后，建议:\n\n1. **创建开发分支**\n   ```powershell\n   git checkout -b develop\n   git push -u origin develop\n   ```\n\n2. **设置GitHub Pages** (可选，用于文档)\n   - 在Settings中启用GitHub Pages\n   - 选择docs文件夹或main分支\n\n3. **定期提交**\n   ```powershell\n   git status\n   git add .\n   git commit -m \"Description of changes\"\n   git push origin main\n   ```\n\n## 参考链接\n\n- [Git官方文档](https://git-scm.com/doc)\n- [GitHub帮助](https://docs.github.com/en/get-started)\n- [SSH密钥配置](https://docs.github.com/en/authentication/connecting-to-github-with-ssh)\n- [Personal Access Token](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/creating-a-personal-access-token)\n\n---\n\n**需要帮助?**\n\n如果遇到任何问题，请检查:\n1.
